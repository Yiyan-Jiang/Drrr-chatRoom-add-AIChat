# 接口文档整理

本文档描述后端所有 REST API 接口。Socket.IO 接口详见 `backend/routers/socket.py` 及 `前端实现建议-React.md`。

**基础 URL**：`http://127.0.0.1:8000`（开发环境）
**API 前缀**：`/api`

**认证**：除注册、登录接口外，所有接口需在 Header 中携带 `Authorization: Bearer <access_token>`，token 通过 `/api/auth/login` 获取。

---

## 1. 用户 (users)

### 1.1 注册用户
`POST /api/users/register`

**请求体**：
```json
{
  "username": "string",
  "password": "string"
}
```

**响应**：`201 Created`
```json
{
  "id": 1,
  "username": "testuser",
  "password": "hashed_password",
  "created_at": "2025-01-01T00:00:00"
}
```

**错误**：
- `400 Bad Request`：用户名已存在

### 1.2 按用户名查询用户
`GET /api/users/username/{username}`

**响应**：`200 OK`
```json
{
  "id": 1,
  "username": "testuser",
  "password": "hashed_password",
  "created_at": "2025-01-01T00:00:00"
}
```

**错误**：
- `404 Not Found`：用户不存在

### 1.3 按用户ID查询用户
`GET /api/users/{user_id}`

**响应**：同 1.2

### 1.4 更新用户信息
`PUT /api/users/{user_id}`

**请求体**：
```json
{
  "username": "new_username",
  "password": "new_password"
}
```
字段均可选。

**响应**：`200 OK`（更新后的用户信息）

**错误**：
- `400 Bad Request`：用户名已存在
- `404 Not Found`：用户不存在

### 1.5 删除用户
`DELETE /api/users/{user_id}`

**响应**：`204 No Content`

**错误**：
- `404 Not Found`：用户不存在

## 2. 认证 (auth)

### 2.1 登录
`POST /api/auth/login`

**请求体**：
```json
{
  "username": "string",
  "password": "string"
}
```

**响应**：`200 OK`
```json
{
  "access_token": "eyJ...",
  "token_type": "bearer",
  "expires_in": 3600,
  "user": {
    "id": 1,
    "username": "testuser",
    "created_at": "2025-01-01T00:00:00"
  }
}
```

**错误**：
- `401 Unauthorized`：用户名或密码错误

**说明**：
- `expires_in`：token 有效秒数，默认 3600 秒（1小时）
- 此 token 用于后续 API 认证及 Socket.IO 连接（`auth: { token }`）

## 3. 房间 (rooms)

### 3.1 创建房间
`POST /api/rooms/`

**请求体**：
```json
{
  "name": "房间名称"
}
```

**响应**：`201 Created`
```json
{
  "id": 1,
  "name": "房间名称",
  "created_at": "2025-01-01T00:00:00"
}
```

**错误**：
- `400 Bad Request`：房间名已存在

### 3.2 获取房间列表
`GET /api/rooms/`

**查询参数**：
- `skip`：跳过条数（默认 0）
- `limit`：返回条数（默认 50，最大 200）

**响应**：`200 OK`
```json
[
  {
    "id": 1,
    "name": "房间1",
    "created_at": "2025-01-01T00:00:00"
  }
]
```

### 3.3 按名称查询房间
`GET /api/rooms/name/{name}`

**响应**：`200 OK`（同 3.1 响应格式）

**错误**：
- `404 Not Found`：房间不存在

### 3.4 获取房间详情（含最近消息）
`GET /api/rooms/{room_id}`

**响应**：`200 OK`
```json
{
  "id": 1,
  "name": "房间名称",
  "created_at": "2025-01-01T00:00:00",
  "messages": [
    {
      "id": 1,
      "content": "消息内容",
      "user_id": 1,
      "room_id": 1,
      "created_at": "2025-01-01T00:00:00"
    }
  ]
}
```

**错误**：
- `404 Not Found`：房间不存在

### 3.5 删除房间
`DELETE /api/rooms/{room_id}`

**响应**：`204 No Content`

**错误**：
- `404 Not Found`：房间不存在

## 4. 消息 (messages)

> 注：实时消息收发通过 Socket.IO，此 HTTP 接口仅作辅助。

### 4.1 获取房间消息
`GET /api/messages/?room_id={room_id}`

**查询参数**：
- `room_id`：房间ID（必需）

**响应**：`200 OK`
```json
[
  {
    "id": 1,
    "content": "消息内容",
    "user_id": 1,
    "room_id": 1,
    "created_at": "2025-01-01T00:00:00"
  }
]
```

### 4.2 发送消息
`POST /api/messages/`

**请求体**：
```json
{
  "content": "消息内容",
  "room_id": 1
}
```

**响应**：`201 Created`（创建的消息）

**错误**：
- `400 Bad Request`：房间不存在

### 4.3 删除消息
`DELETE /api/messages/{message_id}`

**响应**：`204 No Content`

**错误**：
- `404 Not Found`：消息不存在

## 5. AI 聊天 (ai)

### 5.1 AI 聊天（SSE 流式）
`POST /api/ai/chat`

**Header**：
- `Authorization: Bearer <access_token>`（必需）

**请求体**：
```json
{
  "message": "用户消息",
  "character": "sakura"  // 可选：sakura/rin/mio，默认 sakura
}
```

**限制**：
- `message`：1-500字符
- `character`：可选，默认 `sakura`，可选值 `sakura`（傲娇小樱）、`rin`（三无凛）、`mio`（病娇澪）

**响应**：`200 OK`，`Content-Type: text/event-stream`

**SSE 格式**：
```
data: AI回复片段1

data: AI回复片段2

data: [DONE]
```

**错误**：
- `401 Unauthorized`：未提供或无效的 token
- SSE 流中可能返回：`data: 错误: 错误信息`

**说明**：
- 使用 **Server-Sent Events (SSE)** 流式返回 AI 回复
- 每次调用会带上用户当前性格的最近 8 条对话历史，保持上下文
- 不同 `character` 的历史记录分开存储，互不影响
- AI 性格由 `character` 参数决定：
  - `sakura`：「傲娇系猫耳女仆小樱」
  - `rin`：「白峰凛 - 三无少女」
  - `mio`：「樱井澪 - 病娇少女」
- 回复内容自动保存到数据库，供后续对话使用

### 5.2 清理 AI 对话历史
`DELETE /api/ai/chat/history?character=sakura`

**Header**：
- `Authorization: Bearer <access_token>`（必需）

**Query 参数**：
- `character`：可选，默认 `sakura`，指定要清理哪个性格的历史

**响应**：`200 OK`
```json
{
  "message": "已清理 X 条历史记录 (sakura)"
}
```

**说明**：
- 清理当前用户指定性格的 **7 天前** 的 AI 对话历史
- 不同性格的历史分开清理，互不影响

## 6. Socket.IO

连接 URL：同基础 URL（`http://127.0.0.1:8000`）

### 6.1 连接认证
```javascript
const socket = io(URL, {
  auth: { token: "access_token" }
});
```

### 6.2 客户端事件
| 事件 | 载荷 | 说明 |
|------|------|------|
| `join_room` | `{ room_id: number }` | 加入房间 |
| `send_message` | `{ room_id: number, content: string }` | 发送消息 |
| `leave_room` | `{ room_id: number }` | 离开房间 |

### 6.3 服务端事件
| 事件 | 载荷 | 说明 |
|------|------|------|
| `previous_messages` | `ChatMessagePayload[]` | 进入房间后的历史消息 |
| `new_message` | `ChatMessagePayload` | 新消息 |
| `user_joined` | `{ user_id: number, room_id: number }` | 其他用户加入房间 |
| `error` | `{ message: string }` | 错误信息 |

### 6.4 消息格式
```typescript
type ChatMessagePayload = {
  id: number;
  content: string;
  user_id: number;
  room_id: number;
  created_at: string | null;
};
```

## 8. 数据模型

### 8.1 用户 (chatRoom_user)
```sql
id INT PRIMARY KEY
username VARCHAR(64) UNIQUE
password VARCHAR(128)
created_at DATETIME
```

### 8.2 房间 (chatRoom_room)
```sql
id INT PRIMARY KEY
name VARCHAR(128) UNIQUE
created_at DATETIME
```

### 8.3 消息 (chatRoom_message)
```sql
id INT PRIMARY KEY
user_id INT FOREIGN KEY (chatRoom_user.id) ON DELETE CASCADE
content TEXT
room_id INT FOREIGN KEY (chatRoom_room.id) ON DELETE SET NULL
created_at DATETIME
```

### 8.4 AI 对话历史 (chatRoom_ai_chat_history)
```sql
id INT PRIMARY KEY
user_id INT FOREIGN KEY (chatRoom_user.id) ON DELETE CASCADE
role VARCHAR(20)  -- 'system', 'user', 'assistant'
content TEXT
created_at DATETIME
INDEX (user_id)
```
