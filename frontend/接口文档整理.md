让 AI 扫的接口文档（懒得补 Apifox 了），已与当前 `backend` 代码对齐；仍建议以 **`/openapi.json`** 为准。

| 项目 | 地址 |
|------|------|
| 服务（默认） | `http://127.0.0.1:8000` |
| Swagger UI | `/docs` |
| ReDoc | `/redoc` |
| OpenAPI JSON | `/openapi.json` |

**前缀**：REST 均在 `/api` 下；Socket.IO 与 FastAPI 挂在同一 ASGI 应用上（根路径连接即可）。

## 1. 认证 `/api/auth`

| 说明 | 方法 | 路径 | 请求 | 成功响应 |
|------|------|------|------|----------|
| 登录（签发短期 access token） | POST | `/login` | Body：`username`、`password`（明文；与注册一致，服务端用 SHA256 hex 与库中摘要比对） | `200` + `LoginResponse` |

**`LoginResponse` 字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| `access_token` | string | 短期令牌；Socket.IO 连接时放在 `auth.token` |
| `token_type` | string | 固定为 `bearer` |
| `expires_in` | int | 有效期秒数（与部署环境配置一致，见下文） |
| `user` | object | `UserPublic`：`id`、`username`、`created_at`（**不含密码**） |

**Token 实现要点**（实现见 `backend/auth_token.py`）：服务端使用 **HMAC-SHA256** 对负载签名；负载含 `sub`（用户 id）、`iat`、`exp`。客户端自行保存 token（内存 / HttpOnly Cookie 等按前端方案）。

**环境变量（可选）**

| 变量 | 说明 | 默认 |
|------|------|------|
| `CHAT_JWT_SECRET` | 签名密钥 | `dev-change-me-in-production`（**生产环境必须修改**） |
| `CHAT_ACCESS_TOKEN_EXPIRE_SECONDS` | 有效期（秒），实际值会被限制在 **60～604800**（7 天） | `3600` |

**常见错误**

- `401`：用户名或密码错误（统一文案：`用户名或密码错误`，避免枚举用户名）。

## 2. 用户 `/api/users`

| 说明 | 方法 | 路径 | 请求 | 成功响应 |
|------|------|------|------|----------|
| 注册 | POST | `/register` | Body：`username`、`password`（明文；服务端会做哈希存储） | `201` + `UserInDB` |
| 按用户名查 | GET | `/username/{username}` | — | `200` + `UserInDB` |
| 按 ID 查 | GET | `/{user_id}` | — | `200` + `UserInDB` |
| 更新 | PUT | `/{user_id}` | Body：`UserUpdate`（`username`、`password` 均可选填） | `200` + `UserInDB` |
| 删除 | DELETE | `/{user_id}` | — | `204`，无 Body |

**路由顺序**：`/username/{username}` 必须在 `/{user_id}` **之前** 注册，否则会把路径段 `username` 当成整数 ID 校验失败。

**常见错误**

- `400`：注册或更新时违反用户名唯一约束（与其它用户冲突）。
- `404`：用户不存在。

## 3. 聊天室 `/api/rooms`

| 说明 | 方法 | 路径 | 请求 | 成功响应 |
|------|------|------|------|----------|
| 创建 | POST | `/` | Body：`name` | `201` + `RoomInDB` |
| 分页列表 | GET | `/` | Query：`skip`（默认 `0`，≥0）、`limit`（默认 `50`，范围 1～200） | `200` + `RoomInDB[]` |
| 按名称查 | GET | `/name/{name}` | — | `200` + `RoomInDB` |
| 详情（含消息） | GET | `/{room_id}` | — | `200` + `RoomWithMessages`（最近 **50** 条消息，按创建时间倒序） |
| 删除 | DELETE | `/{room_id}` | — | `204` |

**路由顺序**：`/name/{name}` 须在 `/{room_id}` **之前**，否则会把名称路径误当作房间 ID。

**常见错误**

- `400`：创建房间时名称与其它房间重复。
- `404`：房间不存在。

## 4. 消息 `/api/messages`

即时收发以 **Socket.IO** 为主；以下为 HTTP 辅助接口。

| 说明 | 方法 | 路径 | 请求 | 成功响应 |
|------|------|------|------|----------|
| 创建 | POST | `/` | Body：`content`、`room_id`；**Query**：`user_id`（发送者用户 ID，必填整数） | `201` + `MessageInDB` |
| 某房间列表 | GET | `/room/{room_id}` | — | `200` + `MessageInDB[]`（最近 50 条，倒序） |
| 单条 | GET | `/{message_id}` | — | `200` + `MessageInDB` |
| 删除 | DELETE | `/{message_id}` | — | `204` |

**路由顺序**：`/room/{room_id}` 须在 `/{message_id}` **之前**。

**常见错误**

- `400`：`room_id` 指向的房间不存在等业务错误。
- `404`：消息不存在。

## 5. Socket.IO

连接地址与 HTTP 同源即可（如 `http://127.0.0.1:8000`），具体以客户端库写法为准。

### 5.1 连接与鉴权

建立连接时必须在 **`auth` 中传入 `token`**：

- `token` 为 **`POST /api/auth/login` 返回的 `access_token`**（字符串）。
- 服务端校验令牌签名、有效期，并确认用户仍存在后写入会话（`save_session`）；**不再**接受仅传 `user_id` 的旧方式。

示例（逻辑层面）：`auth: { token: "<access_token>" }`。

### 5.2 客户端 → 服务端

| 事件 | 载荷要点 | 行为摘要 |
|------|-----------|----------|
| `connect` | `auth.token` 为合法 access token | 失败则断开连接 |
| `disconnect` | — | 断开 |
| `join_room` | `room_id` 须为 **整数类型**（不要用布尔代替数字） | 校验房间存在；进入名为 `room_{room_id}` 的 Socket 房间；向他人发 `user_joined`，向自己发 `previous_messages` |
| `send_message` | `room_id` 同上；`content` 为非空字符串 | 入库并向该 Socket 房间广播 `new_message` |
| `leave_room` | `room_id` 为整数时离开对应 Socket 房间 | — |

### 5.3 服务端 → 客户端

| 事件 | 载荷 |
|------|------|
| `previous_messages` | 消息对象数组：`id`、`content`、`user_id`、`room_id`、`created_at`（ISO 字符串） |
| `new_message` | 单条消息，字段同上 |
| `user_joined` | `user_id`、`room_id`（发给房间内其它连接） |
| `error` | `message`：错误说明字符串 |

## 6. HTTP 状态码约定

| 码 | 典型场景 |
|----|-----------|
| `200` / `201` | 成功 |
| `204` | 删除成功，无响应体 |
| `400` | 业务规则不满足（如唯一约束冲突、房间不存在导致无法发消息等） |
| `401` | 未授权（如登录用户名/密码错误） |
| `404` | 资源不存在 |
| `422` | 路径 / Query / Body 不符合校验规则（FastAPI 默认） |
